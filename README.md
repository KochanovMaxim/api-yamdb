API YaMDb

Описание проекта:
YaMDb (Yet another Movie Database) — это REST API для сбора отзывов пользователей на произведения, такие как фильмы, книги и музыка. Пользователи могут оставлять отзывы, ставить оценки произведениям, а также комментировать отзывы других пользователей. Проект предоставляет полноценную систему аутентификации, авторизации и управления контентом с различными уровнями доступа для разных ролей пользователей.

Основные возможности:
- Аутентификация и авторизация через email с отправкой кода подтверждения
- Управление пользователями с ролями: администратор, модератор, обычный пользователь
- Полный набор CRUD-операций для произведений, категорий и жанров
- Система отзывов и оценок к произведениям
- Комментарии к отзывам других пользователей
- Поиск и фильтрация произведений по различным параметрам
- Пагинация и сортировка результатов

Стек технологий:
- Python 3.8 и выше
- Django 4.0+
- Django REST Framework 3.12+
- PostgreSQL в качестве базы данных
- Simple JWT для аутентификации с использованием JWT-токенов
- Django Filter для реализации фильтрации данных
- Gunicorn для production-развертывания
- Nginx в качестве веб-сервера

Установка и запуск проекта:

1. Клонирование репозитория:
Склонируйте репозиторий с помощью команды git clone https://github.com/KochanovMaxim/api-yamdb.git и перейдите в директорию проекта cd api_yamdb.

2. Создание виртуального окружения:
Создайте виртуальное окружение python -m venv venv и активируйте его:
- Для Linux/Mac: source venv/bin/activate
- Для Windows: venv\Scripts\activate

3. Установка зависимостей:
Установите все необходимые зависимости командой pip install -r requirements.txt.

4. Применение миграций:
Выполните миграции для создания структуры базы данных: python manage.py migrate

5. Заполнение базы данных тестовыми данными:
Проект включает скрипт для заполнения базы данных тестовыми данными из CSV файлов:
python load_csv_data
Или альтернативно можно выполнить:
python manage.py shell
>>> from utils.load_csv_data import main
>>> main()

6. Создание суперпользователя:
Создайте учетную запись администратора командой python manage.py createsuperuser.

7. Запуск сервера:
Запустите сервер разработки командой python manage.py runserver.
Сервер будет доступен по адресу: http://127.0.0.1:8000/

8. Запуск тестов:
Для запуска тестов используйте команду python manage.py test.

Структура проекта:
Проект имеет следующую структуру:
- api/ - основное API приложение с views, serializers, urls и filters
- users/ - приложение для управления пользователями, аутентификации и авторизации
- reviews/ - приложение для работы с произведениями, отзывами и комментариями
- api_yamdb/ - корневая директория с настройками проекта
- requirements.txt - файл с зависимостями проекта
- utils/load_csv_data.py - скрипт для загрузки тестовых данных

Примеры API запросов:

1. Регистрация нового пользователя:
Метод: POST
Эндпоинт: /api/v1/auth/signup/
Тело запроса: {"email": "user@example.com", "username": "username"}
Ответ: {"email": "user@example.com", "username": "username"}

2. Получение JWT-токена:
Метод: POST
Эндпоинт: /api/v1/auth/token/
Тело запроса: {"username": "username", "confirmation_code": "123456"}
Ответ: {"token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."}

3. Получение списка произведений:
Метод: GET
Эндпоинт: /api/v1/titles/
Заголовок: Authorization: Bearer <ваш-токен>
Ответ содержит список произведений с информацией о названии, годе выпуска, рейтинге, описании, жанрах и категории.

Пример ответа:
[
  {
    "id": 1,
    "name": "Пример произведения",
    "year": 2020,
    "rating": 8.5,
    "description": "Описание произведения",
    "genre": [
      {
        "name": "Драма",
        "slug": "drama"
      }
    ],
    "category": {
      "name": "Фильмы",
      "slug": "movies"
    }
  }
]

4. Создание отзыва:
Метод: POST
Эндпоинт: /api/v1/titles/1/reviews/
Заголовок: Authorization: Bearer <ваш-токен>
Тело запроса: {"text": "Отличный фильм!", "score": 9}

5. Получение данных текущего пользователя:
Метод: GET
Эндпоинт: /api/v1/users/me/
Заголовок: Authorization: Bearer <ваш-токен>
Ответ: данные пользователя включая username, email, имя, фамилию, bio и роль.

Роли пользователей:

Администратор (admin):
- Полный доступ ко всем эндпоинтам API
- Может назначать роли другим пользователям
- Может создавать, читать, обновлять и удалять любые объекты
- Имеет доступ к панели администратора Django

Модератор (moderator):
- Может изменять и удалять любые отзывы и комментарии
- Не может управлять пользователями и их ролями
- Не может управлять произведениями, категориями и жанрами

Пользователь (user):
- Может оставлять отзывы и комментарии
- Может редактировать и удалять только свои собственные отзывы и комментарии
- Может редактировать свой собственный профиль
- Не может изменять свою роль
